How to use it

Put your Zoom app credentials in the matching panel, start authorization, then return here after Zoom redirects back. The app will offer the right token exchange for that callback.

Security note

Confidential credentials are stored only in an ephemeral server-side session tied to an HttpOnly cookie. The browser never writes your client secret into local or session storage.

Setup and test plan

Zoom Marketplace build and validation flow

Marketplace setup

  1. Create a General App in the Zoom App Marketplace build flow.
  2. In App Credentials, copy the standard Client ID and Client Secret.
  3. Add http://localhost:3000/ as a redirect URL in development.
  4. Add the scopes you want to test, such as user:read.
  5. Turn on Use Public Client OAuth to get a Public Client ID for PKCE.
  6. Use the Local Test page if you want Zoom’s build flow to add the app to your account before testing.

Validation pass

  1. Run the confidential flow with the normal client ID and client secret.
  2. Run the PKCE flow with the Public Client ID and confirm the code exchange uses code_verifier.
  3. Compare returned scopes, token payloads, and any authorization prompts between both flows.
  4. Repeat after changing scopes or redirect URLs, since Zoom may ask you to re-authorize after those updates.
